# Regra de Comportamento: Atualização Automática do Roadmap

Esta regra é "Always On" e define a obrigação comportamental de manutenção e atualização do cronograma de progresso do projeto.

---

## 📋 Diretriz Contínua
Você (agente) tem a **obrigação comportamental permanente** de manter o arquivo físico `ROADMAP.md` na raiz do projeto sempre atualizado. 

Antes de finalizar qualquer interação ou sugerir a conclusão de uma tarefa ao usuário, você DEVE seguir este fluxo:
1. Ler o arquivo `ROADMAP.md` na raiz do projeto para avaliar o estado geral do cronograma.
2. Identificar quais itens foram implementados, refatorados ou adicionados nesta sessão.
3. Atualizar proativamente o status dos itens no `ROADMAP.md`:
   - Mudar de `[ ]` (pendente) para `[x]` (concluído) ou `[/]` (parcialmente concluído).
4. Se uma funcionalidade for estendida ou nova documentação for criada, registre uma pequena linha no histórico de progresso do roadmap.
5. Nunca entregue a tarefa ao usuário sem realizar este passe de manutenção documental.
